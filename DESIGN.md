# LAZING Design Direction

## Product Feeling

LAZING should not look technical.
It should feel technically powerful.

Outside:

- calm
- white
- glassy
- light
- precise
- human

Inside:

- agents
- skills
- SOPs
- fields
- credentials
- routines
- swarming
- benchmarking
- trace

The interface stays quiet.
The system underneath feels alive.

## Core Thesis

```txt
The system adapts to you.
Not the other way around.
```

The page should avoid generic AI productivity language.
It should explain a new relationship to software:

```txt
I describe what I want to achieve.
The system builds the structure.
I decide.
The system executes.
```

## Apple / Rams Rules

- Use typography for hierarchy, not decoration.
- Keep font weights restrained: regular, medium and semibold before bold.
- Use glass/material effects to create depth and place, not to tint everything.
- Use fewer, stronger modules instead of many equal feature cards.
- Let whitespace create confidence, but do not create empty dead zones.
- Make layout adaptive: desktop can be theatrical, mobile must be direct.
- Controls need stable dimensions and enough touch area.
- Reduce until the remaining objects explain the product.

## Bento Rules

Use bento modules when a section needs rhythm:

- one large thesis block
- one practical example block
- one narrow operational block
- one soft trust/privacy block
- quick-choice pills for action language

Do not make every bento tile identical.
Vary width, height, tone and density.

## Copy Rules

Prefer short, inevitable sentences:

```txt
The interface appears when needed.
Expertise becomes executable.
Every field has its own memory.
Every memory has a boundary.
Many minds. One decision.
```

Avoid:

- AI-powered productivity platform
- unlock your workflow
- seamless collaboration
- supercharge
- all-in-one dashboard

## Page Rhythm

1. Hero: emotional product thesis.
2. Intent: old relationship vs new relationship.
3. Manifestation Layer: interfaces appear from intent.
4. Fields: boundaries, memory and credentials.
5. Pocket Command Center: quick choices and heartbeats.
6. Skills + SOPs: expertise becomes executable.
7. Developers: engines, swarms, trace.
8. Creators: methods, not only content.
9. Trust: scope is product quality.
