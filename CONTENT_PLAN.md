# Lazing Audience and Content Plan

## Positioning Rule

Lazing should not sound like another AI productivity tool.

Use this simple mapping everywhere:

- Codex, Claude Code and Ollama are engines.
- Lazing is the cockpit, scope and trace layer around them.
- A chat answer is not always enough.
- Sometimes the answer should become a small working system.
- The system adapts to the person, not the person to the system.

## Target Audiences

### Codex and Claude Code Users

They already understand powerful AI coding engines. They need a reason to care about Lazing.

Message:

> Your engines are powerful. Lazing gives them scope, memory boundaries and a surface people can actually steer.

Pain:

- context sprawl
- unclear memory
- hard-to-repeat workflows
- local/VPS setup friction
- no clean trace of decisions and actions

Proof to show:

- adapter direction
- local runtime authority
- trace viewer direction
- pack test harness direction
- scoped command center instead of open-ended agent sprawl

### Paperclip, OpenClaw and OSS AI Tool Users

They care about open source, control and shipping useful workflows.

Message:

> Lazing is not trying to be the biggest dashboard. It is trying to make one intention become the right working surface.

Pain:

- generic app shells
- too many panels
- weak product taste
- feature lists without a human story
- unclear trust model

Proof to show:

- premium white Lazing interface
- Manifestation Layer vocabulary
- Founding Circle applications
- no analytics by default
- European privacy direction

### People New To Local AI

They do not care about adapters first. They need a clean analogy.

Message:

> Tell Lazing what you want. It builds the small system around it: the tracker, plan, decision, routine or workspace.

Pain:

- AI feels abstract
- chat feels temporary
- tools feel complicated
- privacy feels unclear

Proof to show:

- phone-first command center
- personal field
- local/VPS runtime
- "private by default" language
- simple examples: meal log, launch plan, client workspace, learning sprint

## German SEO Mapping

Keep the main site English. Use `/de` as the dedicated German-language intent page.

Use natural German, not translated SaaS German:

- "lokales KI-Kommandozentrum"
- "KI Tool lokal nutzen"
- "selbst gehostete KI"
- "private KI Workflows"
- "KI Agenten mit Kontrolle"
- "Claude Code, Codex und Ollama verbinden"
- "Du sagst, was du willst. Lazing baut das System darum."
- "Nicht noch ein Dashboard. Eine ruhige Oberfläche für Absicht, Grenze und Trace."

Avoid:

- "KI-gestützte Produktivitätsplattform"
- "End-to-end Workflow Automation Suite"
- "All-in-one AI Hub"
- overexplaining "Manifestation" before the analogy lands
